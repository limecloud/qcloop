# qcloop v0.6.1

发布日期：2026-05-11

## 亮点

- 本机 AI CLI 执行超时后会显式清理进程树，避免 `codex` / `claude` / `gemini` 等 CLI 派生的后台子进程继续占用资源。
- 执行器超时时间可通过环境变量配置，便于把 CI、交互调试和长任务托管拆成不同预算。

## 行为变更

- Unix 平台运行 CLI 时创建独立进程组，超时或取消时对整个进程组发送终止信号。
- Windows 平台保持 `Process.Kill` 的最小实现，避免引入未验证的作业对象抽象。
- 默认超时仍为 5 分钟；支持 `QCLOOP_AGENT_TIMEOUT`、`QCLOOP_AGENT_TIMEOUT_MS`、`QCLOOP_CODEX_TIMEOUT`、`QCLOOP_CODEX_TIMEOUT_MS`、`QCLOOP_EXECUTOR_TIMEOUT` 与 `QCLOOP_EXECUTOR_TIMEOUT_MS`。
